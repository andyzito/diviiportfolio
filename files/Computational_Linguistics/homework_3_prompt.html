LINGUIST492A: Homework #3

n-gram models

In this assignment, you will train bigram word models. The primary goals of this homework are to implement realistic statistical inference, including various smoothing methods, and evaluate the performance of different smoothing techniques by testing your model's generalization to unseen text.

Part 1: Build a bigram model

Download the Reuters corpus. The Reuters corpus consists of 1.72 million words of text of newswire, spanning 90 different topics (e.g. 'oil' or 'cocoa'). We have provided the entire corpus for you as a list of tokenized words, but we have not removed punctuation or lower-cased any words. Contractions have been split from their host words, and acronyms are represented as many separate words (e.g. [... 'U','.','S','.',...]). These are less than ideal choices, and you may wish to modify them. However, for the purposes of this assignment, you may work with the Reuters corpus as we have provided it; you do not need to remove punctuation, or lower-case, or otherwise modify the corpus if you do not wish to.

In part part, we would like you to write one function:

  • Write a function that takes a list of words as input (for example, the Reuters corpus), and creates a bigram language model based on the data you give it. The bigram model should represent the conditional probably of every word $$$w_i$$$ in the corpus given every other word $$$w_j$$$. That is, your function should return some representation of $$$P(w_i|w_j)$$$ for all values of $$$w_j$$$ and $$$w_i$$$. You may choose to put a start and end symbol at the ends of the corpus, but you do not need to.

Your function should implement Lidstone smoothing, such that the user can pass the smoothing parameter to the function that fits the bigram model to determine how much smoothing they would like.

Advice: We will be testing your function on standardized data, to make sure it yields the correct values! Therefore you want to be very sure it is giving you sensible probabilities before turning it in. But it can be difficult to know if your function is doing something sensible on a corpus so big. We highly recommend you test your function on a very small, toy corpus, small enough to calculate the correct probability values by hand, so you can be sure it's giving you the right values!

Deliverable for Part 1: Provide a .py file that creates the bigram model with Lidstone smoothing. If you decide to do any extra corpus work, please do it outside of the main function that creates a bigram model; we would like to be able to apply your function to any arbitrary list of words.

Part 2: Test your model

Once you have written code that will fit a bigram model, we would like you to test the different smoothing methods.

In this part, we would like you to:

  • Split the Reuters corpus into a training set and a test set. The training set should consist of 2/3 of the corpus, and the test set the remaining 1/3. You may partition the corpus any way that you like.
  • Fit three bigram models: one that uses Laplace, add-one smoothing to estimate probabilities; one that uses Expected Likelihood Estimation; and one that uses Lidstone estimation with a Lidstone parameter of 0.0001.
  • Calculate the cross-entropy and the perplexity of the test data for each of the three models. You may use the cross-entropy and perplexity functions you created from HW#2 if you'd like, or you may create new functions.

Deliverable for Part 2: Turn in a.py script that fits these three models on your training data, and tests them on the test data by calculating the cross entropy and perplexity. In the comments of your script, write up the results of your experiment. At a minimum, we'd like you to: present the cross-entropy / perplexity of each model, and determine which model did the best; describe why you think the best model won; discuss Maximum Likelihood Estimation, and tell us whether you think MLE would've done better with your particular set of training and test data, and why or why not.
